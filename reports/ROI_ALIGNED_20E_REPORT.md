# ROI 对齐版 20 Epoch 对照实验报告

**实验日期**: 2026-03-06  
**实验目的**: 验证 ROI 口径对齐后，Topo 相比 Baseline 的真实表现  
**核心修改**: 训练阶段 Dice loss 改为 ROI 内计算（与验证口径一致）

---

## 1. 实验设置

### 1.1 实验组

| 组名 | 训练脚本 | Dice Loss | Topo Loss | λ策略 |
|:---|:---|:---:|:---:|:---|
| Baseline-ROI | `train_baseline_roi.py` | ROI内 | 无 | N/A |
| Topo-ROI | `train_topo_roi.py` | ROI内 | 有 | 3175 (0.1→0.5) |

### 1.2 关键修改

与原版相比，两组唯一的修改是：**Dice loss 在 ROI 内计算**。

```python
# 原版（全图计算）
loss = self.criterion(outputs, vessels)

# ROI对齐版（ROI内计算）
loss = compute_dice_loss_roi(outputs, vessels, rois)
```

### 1.3 配置参数

- **数据模式**: Kaggle 联合数据集
- **ROI 模式**: fov（基于图像内容估计视场）
- **Epoch**: 20
- **Batch Size**: 4
- **学习率**: 1e-4 (CosineAnnealingLR)
- **Topo Loss**: target_beta0=5, max_death=0.5, loss_scale=100.0
- **随机种子**: 42

## 3. 关键发现

### 3.1 ROI 对齐后，Baseline 与 Topo 的 Val Dice 差距是否缩小？

**没有缩小，反而显著扩大。**

| 对比维度 | 数值 | 说明 |
|:---|:---:|:---|
| **最终 Val Dice 差距** | **-47.4%** | Baseline: 76.9%, Topo: 29.5% |
| 收敛速度差距 | Baseline 快得多 | Baseline 10 epoch 达 71.3%，Topo 仅 25.4% |
| 收敛稳定性 | Baseline 更稳定 | Baseline 曲线平滑，Topo 波动较大 |

**关键观察**:
- ROI 对齐后，Baseline 表现非常出色，20 epoch 达到 **76.9% Dice**
- Topo-ROI 严重欠拟合，20 epoch 仅达到 **29.5% Dice**
- 这表明 **Topo loss 在 ROI 对齐设置下产生了强烈的负面优化效应**

### 3.2 ROI 对齐后，Topo 的 CL-Break 是否比之前更合理？

**没有更合理，反而严重恶化。**

| 指标 | Baseline-ROI | Topo-ROI | 说明 |
|:---|:---:|:---:|:---|
| **CL-Break** | **7.0** | **48.8** | Topo 碎片数是 Baseline 的 **7 倍** |
| **Δβ₀** | **16.6** | **69.6** | Topo 连通分量误差是 Baseline 的 **4.2 倍** |

**关键观察**:
- Baseline-ROI 的拓扑指标非常优秀（CL-Break=7, Δβ₀=16.6）
- Topo-ROI 的拓扑指标严重退化（CL-Break=48.8, Δβ₀=69.6）
- 这说明 **Topo loss 在当前配置下不仅没有改善拓扑，反而破坏了连通性**

### 3.3 这是否说明"ROI 口径错位"是 Topo 退化的重要原因？

**不是。实验结果表明 ROI 口径错位不是 Topo 退化的主要原因。**

**证据**:

1. **假设如果成立，应该观察到的现象**:
   - ROI 对齐后，Topo 应该比 Baseline 更好或接近
   - 因为 Topo loss 专门优化拓扑结构

2. **实际观察到的现象**:
   - ROI 对齐后，Topo 严重劣于 Baseline（47% vs 77%）
   - Topo 的拓扑指标（CL-Break, Δβ₀）反而比 Baseline 差很多

3. **结论**:
   - **ROI 口径错位不是 Topo 退化的主要原因**
   - **Topo loss 本身存在问题**（目标形状、loss scale、或实现方式）
   - ROI 对齐后，问题暴露得更明显

## 4. 深度分析

### 4.1 为什么 Topo-ROI 表现如此差？

**可能原因分析**:

| 可能原因 | 支持证据 | 优先级 |
|:---|:---|:---:|
| **Topo loss 目标形状不匹配** | target_beta0=5 可能不适合血管树 | 高 |
| **Loss scale 过大** | loss_scale=100 可能压制了 Dice loss | 高 |
| **Filtration 方向问题** | 使用 1-prob 作为 filtration 可能不适合 | 中 |
| **ROI 内外梯度冲突** | Topo 只关注 ROI，但 Dice 也在 ROI 内，可能冲突 | 中 |

**关键证据**: 观察 Topo-ROI 的训练过程
- Train Dice Loss (ROI): 从 0.83 缓慢下降到 0.77（20 epoch 仅下降 7%）
- Train Dice (ROI): 从 0.18 缓慢上升到 0.28（20 epoch 仅上升 10%）
- 这表明 **Topo loss 严重干扰了 Dice loss 的优化**

### 4.2 Ratio 分析

| Epoch | Topo Loss Scaled | Dice Loss | Ratio | 说明 |
|:---:|:---:|:---:|:---:|:---|
| 1 | 0.028 | 0.831 | 3.4% | 初始比例适中 |
| 5 | 0.017 | 0.817 | 2.0% | 比例下降 |
| 10 | 0.013 | 0.788 | 1.6% | 比例持续下降 |
| 20 | 0.008 | 0.767 | 1.0% | 比例过低 |

**关键发现**: 
- 虽然 topo_loss_scaled 的绝对值在下降（因为 loss 在下降）
- 但 Dice loss 下降得更慢，说明 Topo loss 的梯度方向与 Dice loss 不一致
- **Topo loss 可能在引导模型走向一个对 Dice 不利的状态**

## 5. 回答核心问题

> **"把 Dice loss 改成 ROI 内计算后，Topo 相比 Baseline 的真实差距到底变成了什么样？"**

### 答案：

**ROI 对齐后，Topo 相比 Baseline 的真实差距是灾难性的：**

| 对比项 | Baseline-ROI | Topo-ROI | 差距 |
|:---|:---:|:---:|:---:|
| Val Dice | **76.9%** | **29.5%** | **-47.4%** |
| CL-Break | **7.0** | **48.8** | **+41.8** |
| Δβ₀ | **16.6** | **69.6** | **+53.0** |

**核心结论**:

1. **ROI 口径对齐不是 Topo 退化的原因**，相反，它暴露了 Topo loss 本身的严重问题

2. **Topo loss 在当前配置（target_beta0=5, loss_scale=100, 3175策略）下：**
   - 严重阻碍模型学习有效特征
   - 导致模型严重欠拟合（29.5% vs 76.9%）
   - 不仅没有改善拓扑，反而大幅恶化拓扑指标

3. **建议的下一步**:
   - 重新审视 Topo loss 的目标形状（target_beta0）
   - 调整 loss scale 或 λ 调度策略
   - 或考虑 Topo loss 的实现是否存在根本性问题
