### 1.2 共同配置

- **数据**: Kaggle 联合数据集

- **Dice Loss**: ROI 内计算

- **Loss Mode**: `fragment_suppress`
- **Loss Scale**: 100.0
- **策略**: 只惩罚除最长条外的 fragments，不奖励变大

## 3. 核心结论

### 3.1 回答核心问题

> **"在修正 PD→loss 映射方向后，拓扑约束是否已经具备接近或超过 Baseline 的实际潜力？"**

**答案：是的，拓扑约束已经具备接近 Baseline 的实际潜力。**

**证据**:

| 对比维度 | 证据 |
|:---|:---|
| **Val Dice 差距** | 仅差 0.78% (0.7618 vs 0.7696)，在可接受范围内 |
| **拓扑指标持平** | CL-Break 6.8 vs 6.7，Δβ₀ 15.6 vs 15.4，证明拓扑约束有效 |
| **持续提升** | 从 30 epoch (0.7515) 到 114 epoch (0.7618)，仍在提升 |
| **收敛趋势** | 100-125 epoch 曲线平稳，接近收敛 |

### 3.2 关键发现

**1. Fragment Suppress 是有效的 Topo loss 策略**
- 相比 Standard 模式 (0.2953) 提升 **+157%**
- 相比 Main Component 模式 (0.6207) 提升 **+22.7%**
- 接近 Baseline 水平 (差距 < 1%)

**2. Topo loss 需要更长的训练时间**
- Baseline-ROI 30 epoch 即收敛
- Topo-Frag-Sup 需要 100+ epoch 才能接近最佳
- 说明拓扑约束的优化是渐进的

**3. 拓扑指标显著改善**
- 相比 Standard 模式 (CL-Break 48.8, Δβ₀ 69.6)
- Fragment Suppress (CL-Break 6.8, Δβ₀ 15.6) 与 Baseline 持平
- 证明碎片抑制策略有效减少了虚假连通分量

### 3.3 与 Baseline-ROI 的差距分析

| 指标 | Baseline | Topo-Frag-Sup | 差距 | 分析 |
|:---|:---:|:---:|:---:|:---|
| Val Dice | 0.7696 | 0.7618 | -0.78% | 差距很小，可能在误差范围内 |
| Val IoU | 0.6276 | 0.6180 | -1.53% | 差距可接受 |
| Val Prec | 0.7948 | 0.7603 | -4.34% | Topo 略低，Recall 略高 |
| Val Rec | 0.7484 | 0.7632 | +1.97% | Topo 更高，说明更多连通区域被检出 |
| CL-Break | 6.7 | 6.8 | +1.5% | 基本持平 |
| Δβ₀ | 15.4 | 15.6 | +1.3% | 基本持平 |

**关键观察**：
- Topo-Frag-Sup 的 Recall 更高 (0.7632 vs 0.7484)，说明拓扑约束帮助检出了更多血管区域
- Precision 略低，可能存在少量假阳性
- 整体 F1 (Dice) 差距仅 0.78%，非常接近

## 5. 建议

### 5.1 立即可行

**Fragment Suppress 已经具备作为默认 Topo loss 的条件**：
- Val Dice 0.7618，与 Baseline 差距 < 1%
- 拓扑指标与 Baseline 持平
- 训练稳定，无明显退化

### 5.2 进一步优化方向

1. **延长训练至 200 epoch**
   - 当前 125 epoch 可能未完全收敛
   - 观察是否能超过 Baseline

2. **微调超参数**
   - `fragment_penalty_factor`: 当前 1.0，尝试 0.5 / 2.0
   - `loss_scale`: 当前 100.0，尝试 50.0 / 200.0

3. **与 Baseline 进行严格对照**
   - 同样跑 125 epoch Baseline-ROI
   - 排除 epoch 数差异的影响

### 5.3 结论

**拓扑约束在修正 PD→loss 映射方向后，已经具备接近 Baseline 的实际潜力，甚至可能在未来优化中超过 Baseline。**
